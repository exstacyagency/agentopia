# Contributing

This repo is being built in small, reviewable steps.

## Branching

- Create a new branch for each change.
- Keep branch names short and descriptive.
- Prefer a `scaffold-*` or `feature-*` prefix when appropriate.

Examples:

- `scaffold-agent-infra-v6`
- `feature-docs-roadmap`

## Pull requests

- Open one PR per branch.
- Keep PRs focused on a single pass.
- Include a short summary of what changed and how it was validated.

## Local workflow

1. Make the change.
2. Run validation scripts.
3. Commit the change.
4. Push the branch.
5. Open the PR.

## Repo rules

- Do not commit secrets.
- Prefer additive changes over rewrites.
- Keep configuration and runtime concerns separated.
